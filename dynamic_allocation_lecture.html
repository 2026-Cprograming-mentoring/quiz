동적 메모리 할당

프로그램 실행 중에 필요한 만큼 메모리를 확보하고, 다 쓰면 반환하는 기술. malloc, calloc, realloc, free의 원리와 사용법을 배웁니다.

📦 <stdlib.h> 🧠 Heap 메모리 🔗 포인터 필수
SECTION 01

왜 동적 할당이 필요한가?

C에서 일반적인 배열은 컴파일 시점에 크기가 결정됩니다. 하지만 실제 프로그램에서는 사용자가 몇 개의 데이터를 입력할지 미리 알 수 없는 경우가 대부분입니다.

problem.c
// ❌ 문제 상황: 학생 수를 미리 알 수 없다
int scores[100];  // 100명 고정? 200명이면? 10명이면 메모리 낭비!

// ✅ 해결: 동적 할당으로 필요한 만큼만 확보
int n;
scanf("%d", &n);
int *scores = (int *)malloc(sizeof(int) * n);  // n명분만 할당!
정적 할당 (Stack)동적 할당 (Heap)
크기 결정컴파일 시점에 고정실행 중에 자유롭게 결정
메모리 위치스택(Stack) 영역힙(Heap) 영역
해제 방식함수 종료 시 자동 해제free()로 수동 해제
크기 변경불가능realloc()으로 가능
사용 예int arr[10];malloc(sizeof(int)*n)
📌 핵심 개념

동적 할당 함수들은 모두 <stdlib.h> 헤더에 선언되어 있습니다. 반드시 #include <stdlib.h>를 추가하세요.

SECTION 02

malloc() — 메모리 할당

malloc(memory allocation)은 지정한 바이트 수만큼 힙 메모리를 할당하고, 그 시작 주소를 반환합니다.

문법
void *malloc(size_t size);  // size: 할당할 바이트 수
                                // 반환: 할당된 메모리의 시작 주소 (실패 시 NULL)

할당할 크기 계산

sizeof(자료형) × 개수로 필요한 바이트 수를 계산합니다.
int 5개 = sizeof(int) × 5 = 4 × 5 = 20바이트

malloc 호출 + 형변환

malloc은 void *를 반환하므로 사용할 포인터 타입으로 캐스팅합니다.

NULL 체크

메모리 부족 시 NULL을 반환합니다. 반드시 확인 후 사용하세요!

malloc_example.c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n;
    printf("학생 수: ");
    scanf("%d", &n);

    // 1단계: int n개 만큼 메모리 할당
    int *scores = (int *)malloc(sizeof(int) * n);

    // 2단계: NULL 체크 (할당 실패 대비)
    if (scores == NULL) {
        printf("메모리 할당 실패!\n");
        return 1;
    }

    // 3단계: 배열처럼 사용
    for (int i = 0; i < n; i++) {
        printf("점수 %d: ", i + 1);
        scanf("%d", &scores[i]);   // *(scores+i) 와 동일
    }

    // 4단계: 사용 후 반드시 해제
    free(scores);
    scores = NULL;  // 안전을 위해 NULL로 초기화

    return 0;
}

📦 메모리 구조 시각화 — malloc(sizeof(int) * 5)

scores →
[0]??
[1]??
[2]??
[3]??
[4]??
↑ 초기값이 쓰레기값임에 주의! (0이 아님)
⚠ 주의: malloc은 초기화하지 않는다!

malloc으로 할당된 메모리에는 쓰레기값(garbage value)이 들어있습니다. 사용 전에 직접 초기화하거나, 초기화가 필요하면 calloc을 사용하세요.

SECTION 03

calloc() — 0으로 초기화된 할당

calloc(contiguous allocation)은 malloc과 동일하게 메모리를 할당하되, 모든 비트를 0으로 초기화해 줍니다.

문법
void *calloc(size_t count, size_t size);
// count: 요소 개수
// size : 각 요소의 크기(바이트)

// 사용 예
int *arr = (int *)calloc(5, sizeof(int));  // int 5개, 모두 0으로 초기화

📦 calloc(5, sizeof(int)) 결과

arr →
[0]0
[1]0
[2]0
[3]0
[4]0
↑ 모두 0으로 초기화됨 ✓
비교malloccalloc
인자malloc(바이트수)calloc(개수, 크기)
초기화안 함 (쓰레기값)0으로 초기화
속도약간 빠름약간 느림 (초기화 때문)
용도직접 초기화할 때0 초기화가 필요할 때
SECTION 04

realloc() — 크기 변경

realloc(re-allocation)은 이미 할당된 메모리의 크기를 늘리거나 줄입니다. 기존 데이터는 유지됩니다.

문법
void *realloc(void *ptr, size_t new_size);
// ptr     : 기존에 할당받은 포인터
// new_size: 새로 원하는 바이트 수
// 반환    : 새 메모리 주소 (기존과 다를 수 있음!)
realloc_example.c
int *arr = (int *)malloc(sizeof(int) * 3);
arr[0] = 10;  arr[1] = 20;  arr[2] = 30;

// 3개 → 5개로 확장
int *temp = (int *)realloc(arr, sizeof(int) * 5);

if (temp != NULL) {
    arr = temp;          // 새 주소로 업데이트
    arr[3] = 40;         // 새로 확장된 공간 사용
    arr[4] = 50;
}

free(arr);

📦 realloc 동작 과정

Before:
[0]10
[1]20
[2]30
↓ realloc(arr, sizeof(int) * 5)
After:
[0]10
[1]20
[2]30
[3]??
[4]??
↑ 기존 데이터는 유지, 새 공간은 초기화 안 됨
🚨 절대 주의: realloc 반환값을 임시 변수로 받자!

만약 arr = realloc(arr, ...)처럼 직접 덮어쓰면, realloc이 실패(NULL 반환)했을 때 기존 메모리 주소를 잃어버려서 메모리 누수가 발생합니다. 반드시 temp 변수로 먼저 받으세요!

SECTION 05

free() — 메모리 해제

동적으로 할당한 메모리는 자동으로 해제되지 않습니다. 반드시 free()로 반환해야 합니다.

free 사용 패턴
int *p = (int *)malloc(sizeof(int) * 10);

// ... p를 사용한 작업 수행 ...

free(p);       // 메모리 해제
p = NULL;      // ★ 댕글링 포인터 방지

free(ptr) 호출

ptr이 가리키는 힙 메모리 블록을 운영체제에 반환합니다.

ptr = NULL 설정

해제된 메모리를 가리키는 포인터를 댕글링 포인터(dangling pointer)라 합니다. NULL로 초기화하여 실수로 접근하는 것을 방지합니다.

SECTION 06

⚠ 자주 하는 실수 & 주의사항

실수 1: 메모리 누수 (Memory Leak)

할당만 하고 free하지 않으면 프로그램이 끝날 때까지 메모리를 잡아먹습니다.

❌ memory_leak.c
void badFunction() {
    int *p = (int *)malloc(sizeof(int) * 100);
    // ... 작업 수행 ...
    return;  // ❌ free 없이 함수 종료! → 메모리 누수
}
실수 2: 이중 해제 (Double Free)

같은 메모리를 두 번 free하면 정의되지 않은 동작(Undefined Behavior)이 발생합니다.

❌ double_free.c
int *p = (int *)malloc(sizeof(int));
free(p);
free(p);  // ❌ 이미 해제된 메모리를 다시 해제! → 크래시 위험

// ✅ 해결: free 후 NULL로 설정하면 free(NULL)은 안전
free(p);
p = NULL;
free(p);  // ✅ free(NULL)은 아무 동작도 안 함
실수 3: 해제 후 접근 (Use After Free)

free한 메모리에 값을 쓰거나 읽으면 프로그램이 비정상적으로 동작합니다.

실수 4: NULL 체크 생략

malloc/calloc/realloc이 NULL을 반환할 수 있음을 잊지 마세요. NULL인 포인터를 역참조하면 세그멘테이션 폴트가 발생합니다.

✅ 안전한 동적 할당 체크리스트

□ #include <stdlib.h> 했는가?

□ 반환값이 NULL인지 확인했는가?

□ 사용이 끝나면 free()를 호출했는가?

□ free 후 포인터를 NULL로 설정했는가?

□ realloc은 임시 변수로 받았는가?

SECTION 07

실전 예제: 동적 배열로 성적 관리

학생 수를 입력받아 동적 배열로 성적을 저장하고, 평균을 계산하는 완전한 프로그램입니다.

grade_manager.c
#include <stdio.h>
#include <stdlib.h>

double calcAverage(int *arr, int size) {
    int sum = 0;
    for (int i = 0; i < size; i++)
        sum += arr[i];
    return (double)sum / size;
}

int main() {
    int n;
    printf("학생 수 입력: ");
    scanf("%d", &n);

    // 동적 할당
    int *scores = (int *)malloc(sizeof(int) * n);
    if (scores == NULL) {
        printf("메모리 할당 실패\n");
        return 1;
    }

    // 입력
    for (int i = 0; i < n; i++) {
        printf("학생 %d 점수: ", i + 1);
        scanf("%d", &scores[i]);
    }

    // 평균 계산 (함수에 동적 배열 전달)
    double avg = calcAverage(scores, n);
    printf("평균: %.1f\n", avg);

    // 추가 학생이 생겼다면? → realloc
    int extra = 2;
    int *temp = (int *)realloc(scores, sizeof(int) * (n + extra));
    if (temp != NULL) {
        scores = temp;
        scores[n] = 95;
        scores[n + 1] = 88;
        n += extra;
        printf("확장 후 평균: %.1f\n", calcAverage(scores, n));
    }

    // 해제
    free(scores);
    scores = NULL;

    return 0;
}

SECTION 09

✓ 이해도 확인 문제

Q1. malloc(20)은 몇 개의 int를 저장할 수 있는가? (sizeof(int) = 4)

① 20개
② 5개
③ 4개
④ 80개
✅ 정답! malloc(20)은 20바이트를 할당합니다. int가 4바이트이므로 20 ÷ 4 = 5개를 저장할 수 있습니다.

Q2. 다음 중 malloc과 calloc의 차이로 올바른 것은?

① calloc이 더 많은 메모리를 할당한다
② calloc은 메모리를 0으로 초기화한다
③ malloc은 스택에, calloc은 힙에 할당한다
④ calloc은 free하지 않아도 된다
✅ 정답! malloc은 쓰레기값이 남아있지만, calloc은 할당된 메모리를 모두 0으로 초기화합니다. 둘 다 힙에 할당하며, 둘 다 반드시 free해야 합니다.

Q3. 아래 코드의 문제점은?

int *p = (int *)malloc(sizeof(int) * 10);
p = (int *)malloc(sizeof(int) * 20); // 크기 변경
free(p);

① 아무 문제 없다
② free를 두 번 호출해야 한다
③ 처음 할당한 10개 공간이 메모리 누수된다
④ realloc을 써야 하므로 컴파일 오류이다
✅ 정답! 두 번째 malloc으로 p가 새 주소를 가리키면서, 처음 malloc으로 받은 10개짜리 메모리의 주소를 잃어버립니다. free할 수 없으므로 메모리 누수! 크기를 변경하려면 realloc을 사용해야 합니다.

Q4. realloc 사용 시 반환값을 임시 변수로 받아야 하는 이유는?

① 속도가 더 빨라서
② 컴파일러가 요구해서
③ 실패 시 NULL이 반환되어 기존 메모리 주소를 잃을 수 있어서
④ realloc은 항상 새 주소를 반환해서
✅ 정답! arr = realloc(arr, ...)에서 realloc이 실패하면 NULL이 arr에 대입되어 기존 메모리의 주소를 잃어버립니다. 그러면 기존 메모리를 free할 수 없어 누수가 발생합니다.
